# 组件 README 生成规范提示词

## 1. 文档目的
生成 widgets 目录下单个组件的 README.md 文件，用于指导开发者正确使用该组件，包含组件功能、使用方法、API 说明及注意事项。

## 2. 文档结构
### 2.1 标题
- 使用组件名称作为一级标题（`# 组件名称`）
- 副标题注明组件所属分类（如：基础组件/业务组件）

### 2.2 基本信息
- **功能描述**：简要说明组件的核心作用和应用场景
- **引入方式**：提供完整的 import 代码示例（如：`import { ComponentName } from '@/widgets/ComponentName'`）
- **兼容性**：说明支持的框架版本、浏览器环境等

### 2.3 API 文档
- **Props 参数**：表格形式列出所有属性，包含字段名、类型、默认值、是否必传、描述
- **事件回调**：表格形式列出所有事件，包含事件名、参数类型、描述
- **插槽（如有）**：说明插槽名称、作用及使用示例

### 2.4 注意事项
- 组件使用限制（如：依赖条件、禁止场景）
- 性能优化建议（如：大数据量渲染注意事项）
- 版本变更记录：以组件内 package.json 中的 version 版本为准，具体描述优先从 git 提交记录中提取，若无法获取则留空（重大更新、breaking change 需重点说明）

## 3. 格式要求
- 使用 Markdown 标准语法，保持排版清晰
- 代码块需指定语言类型（vue/tsx）
- 关键信息使用 **加粗** 或 `代码格式` 突出显示
- 涉及路径使用绝对路径（如：`@/widgets/xxx`）

## 4. 生成要求
- 基于组件源代码自动提取 Props/事件定义，确保 API 准确性
- 示例代码需覆盖主要功能点，逻辑完整可运行
- 语言简洁易懂，避免技术术语堆砌，面向初中级开发者
> **注意**：若历史已生成 README.md 且未进行过修改，重新生成时将保持原文件内容不变，不会覆盖或变更已存在的文档。
## 5. 根目录 README 生成规则
- **内容格式**：仅包含基础组件跳转目录及说明，格式示例：
  ```markdown
  # 组件目录索引
  
  ## 基础组件
  - [组件名称1](./组件名称1/README.md) - 组件1功能说明
  - [组件名称2](./组件名称2/README.md) - 组件2功能说明
  ```
- **更新机制**：新增或删除组件时自动更新目录索引
- **跳转功能**：确保链接可直接跳转至对应组件README.md
